# Политика данных публичного репозитория

Публичный `alterios-mcp` содержит только переиспользуемый код, обезличенные
тесты, общие skills и документацию по контрактам.

## Запрещено публиковать

- адреса реальных экземпляров Alterios и приватных Git-сервисов;
- UUID проектов, форм, представлений, материалов, пользователей и файлов;
- названия, значения и выгрузки материалов реальных проектов;
- raw HAR, screenshots, inventories, write evidence и runtime readback;
- постановки, статусы, комментарии агентов и внутренние задачи заказчика;
- `.env`, tokens, cookies, authorization headers и персональные локальные пути.

## Где хранить проектные материалы

- задачи, постановки и handoff агентов: приватная Gitea;
- inventories, HAR, screenshots, PDF и write journal: локальный каталог
  проекта под `artifacts/` или отдельное приватное хранилище;
- secrets и профили экземпляров: внешний `.env`, который не отслеживается Git.

В публичную документацию переносится только обобщенное правило без реальных
идентификаторов, адресов, имен сущностей и фактических значений.

## Проверка перед push

```powershell
.\.venv\Scripts\python scripts\check_public_tree.py
```

Ошибка проверки блокирует публикацию до удаления project evidence или его
переноса в приватный контур.
