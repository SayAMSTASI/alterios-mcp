# Стандарт иконок Alterios

Этот документ переносит в проект зафиксированные соглашения из `C:\Users\admin\Downloads\standartnye-oboznaceniia.md`, чтобы стандарт иконок был частью репозитория и не терялся между сессиями.

## Базовый стандарт

- использовать только [Google Fonts Icons](https://fonts.google.com/icons);
- базовый размер иконки: `16`;
- базовый цвет иконки: `#4B77D1`.

## Проектный принцип загрузки

- `iconId` в формах, группах и действиях Alterios должен быть UUID файла,
  предварительно загруженного в файловый менеджер конкретного проекта;
- строковые Google names (`save`, `more_vert`, `visibility` и т.д.) используются
  только как исходное имя иконки до загрузки, но не как итоговый `iconId`;
- перед настройкой форм/групп нужно вызвать `alterios_ensure_project_icons` для
  нужного `profile` и `project_id`;
- dry-run `alterios_ensure_project_icons` сохраняет `plan_id`, live-загрузка
  требует `ALTERIOS_MCP_ALLOW_WRITE=1`, `dry_run=false` и тот же `plan_id`;
- результат tool возвращает словарь `semantic -> UUID`, который и надо
  подставлять в `iconId`;
- registry иконок хранится локально в `artifacts/project-icons/<profile>/<project_id>/registry.json`;
  его нельзя считать переносимым между проектами или экземплярами Alterios.

## Инвентаризация иконок файлового менеджера

- `alterios_list_project_icons` читает локальный registry MCP и, если передан
  `folder_hash`, прямые файлы выбранной папки elFinder;
- `folder_hash` можно передавать как из URL браузера (`elf_public_...`) или как
  чистый elFinder hash (`public_...`);
- по умолчанию `icons_folder_name=null` означает "использовать ровно выбранную
  папку", без перехода в подпапки;
- если нужно прочитать подпапку, ее нужно указать явно, например
  `icons_folder_name="icons"`;
- `recurse=false` обязателен для сценария "только текущая папка"; `recurse=true`
  применяется только для осознанной полной инвентаризации дерева;
- `alterios_export_project_icons` скачивает найденные файлы в локальные
  artifacts и формирует `icon-usage-guide.md` с назначением по каждой иконке;
- `alterios_resolve_project_icon` сначала проверяет registry, затем найденные
  файлы проекта, и только потом при необходимости строит guarded upload-plan.

## Общие правила

- иконка выбирается не по вкусу, а по принятому обозначению;
- если для сценария уже есть согласованная иконка, заменять ее на другую нельзя;
- если в исходном стандарте иконка определена визуально, но не названа текстом, нужно использовать именно это согласованное обозначение, а не подбирать ближайший аналог;
- не смешивать Google Icons с другими библиотеками в одном проекте;
- при изменении иконки проверять не только внешний вид, но и смысл действия.

## Принятые обозначения

### Сохранить

- используется во всех формах, где можно изменить данные контента;
- пример сценария: форма редактирования;
- принято располагать в левом нижнем углу после кнопки `Закрыть`;
- ожидаемое поведение: сохранить изменения в форме.

### Назад

- используется во внутренних формах;
- принято располагать первой кнопкой в левом нижнем углу;
- ожидаемое поведение: вернуться на предыдущую форму.

### Редактировать элемент

- используется при показе списка информационных элементов;
- как правило размещается внутри меню действий `more_vert`;
- ожидаемое поведение: открыть форму редактирования значения элемента.

### Просмотр элемента

- используется для открытия формы просмотра значения элемента;
- в большинстве случаев настроена как действие по умолчанию;
- ожидаемое поведение: открыть форму просмотра по кнопке или по нажатию на значение.

### Удалить элемент

- используется для удаления элемента из списка без возможности восстановления;
- ожидаемое поведение: удалить значение элемента.

### Меню действий

- иконка: `more_vert`;
- ожидаемое поведение: раскрыть меню выбора действий.

### Справка или информация

- иконка: `info`;
- подсказка: `Информация/Проверка`;
- ожидаемое поведение: открыть форму со справочной информацией.

### Добавить или создать

- иконка: `add`;
- подсказка: `Добавить/Создать`;
- ожидаемое поведение: открыть форму создания контента.

### Запуск формирования РЗ

- иконка: `experiment`;
- подсказка: описание сути процесса;
- ожидаемое поведение: запустить скрипт формирования рабочих записей в программе испытаний на основе списка НД.

### Мульти-выбор

- используется для сценария множественного добавления контента;
- подсказка: `Мульти-выбор`;
- ожидаемое поведение: запустить процесс множественного добавления контента.

### Синхронизация

- иконка: `directory_sync`;
- подсказка: `Синхронизация`;
- ожидаемое поведение: запустить скрипт синхронизации контента с источником.

### Файлы

- иконка: `attach_file_add`;
- подсказка: `Файлы`;
- ожидаемое поведение: открыть форму с файлами программы или прикрепленными файлами контента.

## Практический стандарт для Codex

- при создании новых действий сначала проверять, есть ли для этого сценария уже принятая иконка в этом документе;
- для `save`, `back`, `edit`, `view`, `delete`, `menu`, `info`, `add`, `sync`, `files` использовать именно эти соглашения;
- если сценарий новый и его нет в списке, сначала описать предлагаемую иконку и согласовать ее, а не молча вводить новый визуальный стандарт.
